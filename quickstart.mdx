## Step 1: Clone the Repository

Clone the FSD Movil repository to your local machine:

<CodeGroup>
```bash HTTPS
git clone https://github.com/elchinito24/fsdmovil.git
cd fsdmovil
```

```bash SSH
git clone git@github.com:elchinito24/fsdmovil.git
cd fsdmovil
```

```bash GitHub CLI
gh repo clone elchinito24/fsdmovil
cd fsdmovil
```
</CodeGroup>

## Step 2: Install Dependencies

Install all required Flutter packages:

```bash
flutter pub get
```

This will install the following key dependencies:

<AccordionGroup>
  <Accordion title="Core Dependencies" icon="cube">

    - **go_router** `^14.6.2`: Declarative routing
    - **dio** `^5.7.0`: HTTP client for API calls
    - **shared_preferences** `^2.3.3`: Local storage for JWT tokens

  <Accordion title="Development Dependencies" icon="code">

  </Accordion>
</AccordionGroup>

<Tip>
  Run `flutter doctor` to verify your Flutter installation and resolve any issues before proceeding.
</Tip>

## Step 3: Configure API Endpoint

FSD Movil needs to know where your backend API is located. There are two ways to configure this:

### Option A: VS Code Launch Configuration (Recommended)

This method keeps your API URL out of version control and makes it easy to switch between environments.

<Steps>
  <Step title="Create launch.json">
    Create the VS Code configuration directory and file:
    
    ```bash
    mkdir -p .vscode
    touch .vscode/launch.json
    ```
  </Step>
  
  <Step title="Add Configuration">
    Open `.vscode/launch.json` and add your configuration:
    
    ```json .vscode/launch.json
    {
      "version": "0.2.0",
      "configurations": [
        {
          "name": "fsdmovil (dev)",
          "request": "launch",
          "type": "dart",
          "toolArgs": [
            "--dart-define=API_URL=https://your-dev-api.com"
          ]
        },
        {
          "name": "fsdmovil (prod)",
          "request": "launch",
          "type": "dart",
          "toolArgs": [
            "--dart-define=API_URL=https://your-prod-api.com"
          ]
        }
      ]
    }
    ```
  </Step>
  
  <Step title="Select Configuration">
    In VS Code, press **F5** or select your configuration from the **Run and Debug** panel
  </Step>
</Steps>

<Warning>
  The `.vscode/launch.json` file is intentionally excluded from Git via `.gitignore`. Never commit API URLs to version control.
</Warning>

### Option B: Command Line Arguments

Pass the API URL directly when running Flutter commands:

```bash
flutter run --dart-define=API_URL=https://your-api.com
```

### How It Works

The API URL is read at compile-time using `String.fromEnvironment`:

```dart lib/config/app_config.dart
class AppConfig {
  static const String baseUrl = String.fromEnvironment(
    'API_URL',
    defaultValue: 'http://10.0.2.2:3000',
  );

  static const int connectTimeout = 10;
  static const int receiveTimeout = 10;
}
```

<Note>
  The default URL `http://10.0.2.2:3000` only works in the Android emulator and points to `localhost:3000` on your development machine.
</Note>

## Step 5: Run the Application

### Using VS Code (Recommended)

<Steps>
  <Step title="Open Project">
    Open the `fsdmovil` folder in VS Code
  </Step>
  
  <Step title="Select Device">
    Click the device selector in the bottom-right corner of VS Code
  </Step>
  
  <Step title="Start Debugging">
    Press **F5** or select **Run > Start Debugging**
  </Step>
  
  <Step title="Choose Configuration">
    Select either "fsdmovil (dev)" or "fsdmovil (prod)" from the dropdown
  </Step>
</Steps>

VS Code will compile the app with your API URL and launch it on the selected device.

### Using Terminal

Run the app directly from the command line:

```bash
flutter run --dart-define=API_URL=https://your-api.com
```

<Note>
  **Hot Reload** is enabled by default. Press **r** in the terminal to reload, **R** to restart, or **q** to quit.
</Note>

  <Step title="Automatic Login">
    You'll be automatically logged in and redirected to the dashboard

### Login with Existing Account

If you already have an account:

```dart
// Authentication happens through the auth provider
final authNotifier = ref.read(authProvider.notifier);
final error = await authNotifier.login(email, password);

if (error == null) {
  // Login successful, JWT token saved
  context.go('/dashboard');
}
```

<Tip>
  JWT tokens are automatically persisted using `shared_preferences`, so you'll stay logged in across app restarts.
</Tip>

## Development Workflow

### Hot Reload

Make changes to your code and see them instantly:

<Tabs>
  <Tab title="VS Code">
    - **Save File**: Press **Ctrl+S** (Windows/Linux) or **Cmd+S** (Mac)
    - Hot reload happens automatically
  </Tab>
  
  <Tab title="Terminal">
    - Press **r** to hot reload
    - Press **R** to hot restart (full restart with state reset)
  </Tab>
</Tabs>

<Warning>
  Hot reload preserves app state but may not reflect changes to:
  - `main()` function
  - Global variables
  - Static fields
  
  Use hot **restart** (R) for these changes.
</Warning>

## Common Issues

<AccordionGroup>
  <Accordion title="API Connection Failed" icon="circle-exclamation">
    **Problem**: App can't connect to backend API
    
    **Solutions**:
    - Verify your API URL is correct in `launch.json` or command line
    - Check that the backend server is running
    - On Android emulator, use `http://10.0.2.2:PORT` for localhost
    - Ensure no firewall is blocking the connection
    - Check network permissions in platform-specific manifests
  </Accordion>
  
  <Accordion title="Dependencies Installation Failed" icon="box">
    **Problem**: `flutter pub get` fails or shows errors
    
    **Solutions**:
    - Run `flutter clean` then `flutter pub get` again
    - Check your internet connection
    - Verify Flutter SDK version: `flutter --version`
    - Delete `pubspec.lock` and try again
    - Check for version conflicts in `pubspec.yaml`
  </Accordion>
  
  <Accordion title="Device Not Detected" icon="mobile-screen">
    **Problem**: Target device doesn't appear in `flutter devices`
    
    **Solutions**:
    - **Android**: Enable USB debugging in Developer Options
    - **iOS**: Trust the computer on your device
    - **Emulator**: Ensure emulator is running before `flutter devices`
    - **Web**: Chrome must be installed
    - Run `flutter doctor` to diagnose platform setup issues
  </Accordion>
  
  <Accordion title="Build Errors After Git Pull" icon="code-branch">
    **Problem**: App fails to build after pulling latest changes
    
    **Solutions**:
    1. Clean the build cache:
       ```bash
       flutter clean
       ```
    2. Re-fetch dependencies:
       ```bash
       flutter pub get
       ```
    3. Rebuild the app:
       ```bash
       flutter run --dart-define=API_URL=https://your-api.com
       ```
  </Accordion>
</AccordionGroup>
