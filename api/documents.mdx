## Overview

Documents represent SRS (Software Requirements Specification) documentation within projects. Each document supports:

- Structured content with requirements
- Version control through revisions
- Collaboration with approval workflows
- Export to DOCX format
- Automatic diagram generation (Mermaid)

<Note>
All document endpoints require authentication via the `Authorization: Bearer {token}` header.
</Note>

---

## GET /documents

Retrieve all documents the authenticated user has access to across all projects.

### Request

No request parameters required. Returns documents based on user's workspace/project access.

### Response

Returns an array of document objects.

<ResponseField name="id" type="string" required>
  Unique document identifier
</ResponseField>

<ResponseField name="title" type="string" required>
  Document title
</ResponseField>

<ResponseField name="version" type="string" required>
  Current document version (e.g., "1.0.0")
</ResponseField>

<ResponseField name="status" type="string" required>
  Document status: `draft`, `review`, `approved`, or `published`
</ResponseField>

<ResponseField name="projectId" type="string" required>
  Identifier of the project this document belongs to
</ResponseField>

<ResponseField name="projectName" type="string" required>
  Name of the project this document belongs to
</ResponseField>

<ResponseField name="workspaceId" type="string" required>
  Identifier of the workspace (via project)
</ResponseField>

<ResponseField name="requirementCount" type="integer" required>
  Number of requirements in this document
</ResponseField>

<ResponseField name="revisionCount" type="integer" required>
  Number of revisions (version history)
</ResponseField>

<ResponseField name="lastModifiedBy" type="string" required>
  Name of user who last modified the document
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of document creation
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  ISO 8601 timestamp of last document update
</ResponseField>

## GET /documents/:id

Retrieve detailed information about a specific document including its content and requirements.

<ParamField path="id" type="string" required>
  Document identifier
</ParamField>

### Request Example


</ResponseField>

## GET /documents/:id/revisions

Retrieve the revision history for a specific document, enabling version control and audit trails.

### Path Parameters

<ParamField path="id" type="string" required>
  Document identifier
</ParamField>

### Request Example

```
GET /documents/doc_1/revisions
```

### Response

Returns an array of revision objects, ordered from newest to oldest.

<ResponseField name="id" type="string" required>
  Unique revision identifier
</ResponseField>

<ResponseField name="documentId" type="string" required>
  Identifier of the parent document
</ResponseField>

<ResponseField name="version" type="string" required>
  Version number of this revision (e.g., "1.0.0")
</ResponseField>

<ResponseField name="changeDescription" type="string" required>
  Description of changes made in this revision
</ResponseField>

<ResponseField name="changedBy" type="object" required>
  Information about the user who created this revision
  
  <Expandable title="User Object Structure">
    <ResponseField name="id" type="string" required>
      User identifier
    </ResponseField>
    
    <ResponseField name="firstName" type="string" required>
      User's first name
    </ResponseField>
    
    <ResponseField name="lastName" type="string" required>
      User's last name
    </ResponseField>
    
    <ResponseField name="email" type="string" required>
      User's email address
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="status" type="string" required>
  Status at time of this revision: `draft`, `review`, `approved`, or `published`
</ResponseField>

<ResponseField name="requirementCount" type="integer" required>
  Number of requirements in this revision
</ResponseField>

<ResponseField name="changesCount" type="object" required>
  Summary of changes in this revision
  
  <Expandable title="Changes Count Structure">
    <ResponseField name="added" type="integer" required>
      Number of requirements added
    </ResponseField>
    
    <ResponseField name="modified" type="integer" required>
      Number of requirements modified
    </ResponseField>
    
    <ResponseField name="removed" type="integer" required>
      Number of requirements removed
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp when this revision was created
</ResponseField>
