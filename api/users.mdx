## Overview

User endpoints provide access to user profiles and management functionality. Users are the foundation of the FSD Movil platform and participate in workspaces, projects, and document collaboration.

<Note>
All user endpoints require authentication via the `Authorization: Bearer {token}` header.
</Note>

---

## GET /users

Retrieve a list of users. The results are typically filtered based on the authenticated user's workspace memberships.

### Request

No request parameters required. Returns users that share workspace access with the authenticated user.

### Response

Returns an array of user objects.

<ResponseField name="id" type="string" required>
  Unique user identifier
</ResponseField>

<ResponseField name="firstName" type="string" required>
  User's first name
</ResponseField>

<ResponseField name="lastName" type="string" required>
  User's last name
</ResponseField>

<ResponseField name="email" type="string" required>
  User's email address
</ResponseField>

<ResponseField name="workspaceCount" type="integer" required>
  Number of workspaces the user belongs to
</ResponseField>

<ResponseField name="projectCount" type="integer" required>
  Number of projects the user has contributed to
</ResponseField>

<ResponseField name="lastActive" type="string" required>
  ISO 8601 timestamp of user's last activity
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of account creation
</ResponseField>

## GET /users/:id

Retrieve detailed information about a specific user including their workspace memberships and recent activity.

### Path Parameters

<ParamField path="id" type="string" required>
  User identifier
</ParamField>

### Request Example

```
GET /users/user_123
```

### Response

<ResponseField name="id" type="string" required>
  Unique user identifier
</ResponseField>

<ResponseField name="firstName" type="string" required>
  User's first name
</ResponseField>

<ResponseField name="lastName" type="string" required>
  User's last name
</ResponseField>

<ResponseField name="email" type="string" required>
  User's email address
</ResponseField>

<ResponseField name="workspaces" type="array" required>
  Array of workspaces the user belongs to
  
  <Expandable title="Workspace Object Structure">
    <ResponseField name="id" type="string" required>
      Workspace identifier
    </ResponseField>
    
    <ResponseField name="name" type="string" required>
      Workspace name
    </ResponseField>
    
    <ResponseField name="role" type="string" required>
      User's role in this workspace: `administrator`, `editor`, or `viewer`
    </ResponseField>
    
    <ResponseField name="joinedAt" type="string" required>
      ISO 8601 timestamp when user joined this workspace
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="recentProjects" type="array" required>
  Array of projects the user recently worked on
  
  <Expandable title="Project Object Structure">
    <ResponseField name="id" type="string" required>
      Project identifier
    </ResponseField>
    
    <ResponseField name="name" type="string" required>
      Project name
    </ResponseField>
    
    <ResponseField name="workspaceId" type="string" required>
      Workspace this project belongs to
    </ResponseField>
    
    <ResponseField name="workspaceName" type="string" required>
      Name of workspace
    </ResponseField>
    
    <ResponseField name="lastActivity" type="string" required>
      ISO 8601 timestamp of user's last activity in this project
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="recentDocuments" type="array" required>
  Array of documents the user recently modified
  
  <Expandable title="Document Object Structure">
    <ResponseField name="id" type="string" required>
      Document identifier
    </ResponseField>
    
    <ResponseField name="title" type="string" required>
      Document title
    </ResponseField>
    
    <ResponseField name="projectId" type="string" required>
      Project this document belongs to
    </ResponseField>
    
    <ResponseField name="projectName" type="string" required>
      Name of project
    </ResponseField>
    
    <ResponseField name="action" type="string" required>
      Last action performed: `created`, `updated`, `approved`, or `published`
    </ResponseField>
    
    <ResponseField name="activityAt" type="string" required>
      ISO 8601 timestamp of the activity
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="statistics" type="object" required>
  User activity statistics
  
  <Expandable title="Statistics Object Structure">
    <ResponseField name="workspaceCount" type="integer" required>
      Total number of workspaces
    </ResponseField>
    
    <ResponseField name="projectCount" type="integer" required>
      Total number of projects contributed to
    </ResponseField>
    
    <ResponseField name="documentCount" type="integer" required>
      Total number of documents created or modified
    </ResponseField>
    
    <ResponseField name="requirementCount" type="integer" required>
      Total number of requirements authored
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="lastActive" type="string" required>
  ISO 8601 timestamp of user's last activity
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of account creation
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  ISO 8601 timestamp of last profile update
</ResponseField>

<Info>
User detail endpoints only return information about users who share at least one workspace with the authenticated user. This ensures privacy and prevents unauthorized user enumeration.
</Info>
